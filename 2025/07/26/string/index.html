计算 π\piπ

怎么优化？

随几个串串，把 π\piπ 算算，把规律看看，发现相邻的 π\piπ 最多增加 1，由此我们可以把长度的枚举上界从 iii 变成 π[i−1]+1\pi[i-1]+1π[i−1]+1，这样最好情况下比较的次数是 n−1n-1n−1，而每次最多加一的性质保障增长的比较次数是 n−2n-2n−2，总比较次数就是 Θ(n)\Theta(n)Θ(n) 的，每次比较时间复杂度为 O(n)O(n)O(n)，所以总的是 O(n2)O(n^2)O(n2)。

自动机

OI 中所说的“自动机”一般都指“确定有限状态自动机”，也就是 dfa\text{dfa}dfa，是计算机科学中常见的数学模型，对于学习 kmp\text{kmp}kmp 等算法有帮助理解的作用。

dfa 是什么

感性理解，dfa\text{dfa}dfa 是一个有向图，用来识别信号序列。信号序列指的是一连串有序的信号。

形式化地，一个 dfa\text{dfa}dfa 由一下五部分组成：

  1. 字符集 Σ\SigmaΣ，该自动机只能输入这些字符。
  2. 状态集合 QQQ。如果把 dfa\text{dfa}dfa 看成一个有向图，那么一个状态就是有向图上的一个顶点。
  3. 起始状态 start，start∈Qstart，start\in Qstart，start∈Q。可以理解为有向图上入度为 0 的一个点。
  4. 接受状态集合 FFF。可以理解为有向图上出度为 0 的一组点。
  5. 转移函数 δ\deltaδ。当前状态是如何到下一个状态的，可以理解为有向图上的边。

如果 dfa\text{dfa}dfa AAA 能识别字符串 SSS，那么 A(S)=TrueA(S)=\text{True}A(S)=True，否则 A(S)=FalseA(S)=\text{False}A(S)=False。

如果一个 dfa\text{dfa}dfa 读入一个字符串是，这个字符串沿着它走能走到接受状态集合，那么说这个字符串被这个 dfa\text{dfa}dfa 接受，否则说这个 dfa\text{dfa}dfa 不接受 这个字符串。

如果一个状态 vvv 没有字符 ccc 的转移，那么令 δ(v,c)=null\delta(v,c)=\text{null}δ(v,c)=null，而 null\text{null}null 只能转移到 null\text{null}null。

我们扩展定义 δ\deltaδ，令 δ(v,s)=δ(δ(v,s[0]),s[1…n−1])\delta(v,s)=\delta(\delta(v,s[0]),s[1\dots n-1])δ(v,s)=δ(δ(v,s[0]),s[1…n−1])，那么 A(s)=[δ(start,s)∈F]A(s)=[\delta(start,s)\in F]A(s)=[δ(start,s)∈F]。

dfa\text{dfa}dfa 的一个好处就是一个串前面的所有信息都可以被一个节点概括，于是可以做 dp\text{dp}dp 的时候把 dfa\text{dfa}dfa 的状态作为状态中的一维。

dfa 的笛卡尔积

如果有两个 dfa\text{dfa}dfa，我们可以构造一个状态集合为两个 dfa\text{dfa}dfa 的状态集合的笛卡尔积，它的每个状态表示“在两个 dfa\text{dfa}dfa 上分别在某个状态”，转移在两个 dfa\text{dfa}dfa 上分别维护。

nfa\text{nfa}nfa 是指可以有多个相同的转移的 dfa\text{dfa}dfa，也就是说走到这里会有好几个分支，只要一个到了接收状态，那这个串就是被接受的。给定 nfa\text{nfa}nfa，我们可以唯一确定一个状态数为 2n2^n2n 的 dfa\text{dfa}dfa。

dfa 最小化

这里的最小化指的是规模上的最小化。

如果两个状态的所有转移都是对应相同的，称这两个状态等价。我们尝试把这两个状态合并起来。

朴素的做法就是把每个状态的转移 Hash\text{Hash}Hash 起来，然后不断合并，直到不能合并。时间复杂度 O(n2Σ)O(n^2\Sigma)O(n2Σ)。

考虑一个更快的方法。既然合并慢，那就分裂。我们在一开始把接受状态合并起来，把拒绝状态合并起来，考虑如果两个状态的同一转移指向了不同的等价类，那么就把这两个状态分裂开。这个算法叫做 moore\text{moore}moore 算法，期望复杂度 O(n(Σ+log⁡n))O(n(\Sigma+\log n))O(n(Σ+logn))，但是如果 dfa\text{dfa}dfa 是一条链，构造使得这些点顺次分裂，每次检查就是 O(n2Σ)O(n^2\Sigma)O(n2Σ) 的。

类似于 SPFA 优化 Bellman-Ford，我们维护一个队列，把每一次分裂后新增的等价类放进去。关键的优化是，我们把一个等价类 xxx 分裂成 y,zy,zy,z 时，可以直接把 y,zy,zy,z 中较小的放进去，另外一个扔掉。这是因为现有的等价类肯定不会被 xxx 分裂了，所以每个等价类到 xxx 的情况都是相同的，若能分裂，只用判 y,zy,zy,z 中任意一个是否一样就行了，不影响正确性，于是时间复杂度就是 O(nΣlog⁡n)O(n\Sigma\log n)O(nΣlogn) 的。

DC3 Algorithm

参考了 应该是发现者的论文。

DC3 Algorithm 是用来求后缀数组的 O(n)O(n)O(n) 算法，跑得要比倍增 O(nlog⁡n)O(n\log n)O(nlogn) 快，但是常数较大并且不好实现。

自然的想法是把串二分，但是最好也只是 O(nlog⁡n)O(n\log n)O(nlogn)，于是考虑另外一种分治方法。将起始坐标 i mod 3=0i\bmod 3=0imod3=0 的归为一类，称为 A 类后缀，其余的归为另一类，称为 B 类后缀。整个的思路是先排 B 类后缀，然后排 A 类后缀，最后归并起来。

第一步，求解 B 类后缀，对于每个后缀用前三个字符进行比较，也就是做三轮基数排序，如果比出来了就结束，否则用以 iii 开头的 6 个字符进行比较。生做肯定会爆掉，于是把字符串重排，把 i mod 3=1i\bmod 3=1imod3=1 的按排出来的顺序拼在一起，中间加上一个 0 隔开，然后把 i mod 3=2i\bmod 3=2imod3=2 的按排出来的顺序拼在一起，接在新串最后，然后递归计算，这样就可以六个六个排了。

第二步，求解 A 类后缀，发现 A 类后缀是由一个 A 类字符和一个 B 类后缀构成的，于是以当前字符为第一关键字，B 类后缀为第二关键字再做基数排序就行了。

第三步，将 A 与 B 归并起来。假设归并到 A 的下标为 iii，B 的下标为 jjj，如果 s[i]≠s[j]s[i]\not=s[j]s[i]=s[j]，那很愉快，否则分类讨论，如果 j mod 3=1j\bmod 3=1jmod3=1，那么 j+1j+1j+1 与 i+1i+1i+1 都是 B 类，它们的顺序我们已经求过；如果 j mod 3=2j\bmod 3=2jmod3=2，那么 j+1j+1j+1 与 i+1i+1i+1 的类别正好反过来，可以转化为上一种情况。

这样通过巧妙的三步，我们就可以求出 sasasa。
